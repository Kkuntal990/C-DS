\section{Problem}
(Q 48) Consider the following C function.
\begin{lstlisting}
    int tob(int b, int *arr){
        int i;
        for (int i = 0; b > 0; i++){
            if(b%2)
                arr[i] = 1;
            else
                arr[i] = 0;
            b = b / 2;
        }
    
        return (i);
    }
\end{lstlisting}

\begin{lstlisting}
    int pp(int a, int b){
    int arr[20];
    int i, tot = 1, ex, len;
    ex = a;

    len = tob(b, arr);
    for (int i = 0; i < len; i++)
    {
        if(arr[i] == 1)
            tot = tot * ex;
        ex = ex * ex;
    }

    return tot;
}
\end{lstlisting}

The value returned by $pp(3,4)$ is ?

\section{Solution}
\textbf{$$pp(3,4) = 81$$}
\newline
\textbf{Explanation}
\newline
Characteristics of \textsl{tob} function:
\begin{enumerate}
    \item If $b \geq 2^{(length(arr))}$, \textsl{tob} returns an error due to buffer overflow.
    \item Converts positive integers to their binary representation.
    \item In the case of negative integer, it returns 1 as output.
\end{enumerate}

\[ tob((b)_{10}, *arr) = 
    \begin{cases} 
    1 & x < 0 \\
    (b)_2 & 0 \geq b < 2^{length(arr)} \\
    undefined & b \geq 2^{length(arr)}
 \end{cases}
\]

For eg. $$\textsl{tob(4, *arr)} = 100 $$, where *arr points to arr[20].

We have $a = 3$ and $b = 4$ and we have $len(arr) = 20$
$$\implies b < 2^{20} $$
Final answer is updated in function \textsl{pp} only when the corresponding bit is set in
binary representation of \textsl{b} as evident from the below for loop.

 \begin{lstlisting}
    for (int i = 0; i < len; i++)
    {
        if(arr[i] == 1)
            tot = tot * ex;
        ex = ex * ex;
    }
 \end{lstlisting}
